# sseemo — 要件定義書 v0.5

S3互換ストレージを使った個人向けE2E暗号化バックアップアプリ

---

## 1. プロダクト概要

個人向けのE2E暗号化バックアップアプリ。ユーザーが選んだS3互換ストレージ、もしくは運営が用意したストレージにファイル・写真・書類を安全に保管する。

**信頼の核**：鍵さえあればアプリなしでも復号可能な設計。サブスク失効・アプリ停止・運営消滅でもデータは救出できる。

---

## 2. 配信・プラットフォーム

- iOSアプリ（App Store配信）
- React Native で実装
- 課金は App Store IAP

---

## 3. ストレージモード

| | マネージドモード | BYOモード |
|---|---|---|
| バケット提供 | 運営 | ユーザー指定（S3互換なら何でも） |
| ストレージ料金 | アプリ料金に内包 | ユーザーがプロバイダに直接支払い |
| 無料枠（アカウント合算） | 5GB | 10GB |
| 暗号化 | E2E（運営も復号不可） | E2E（運営も復号不可） |

---

## 4. 課金

- **¥480 / 月 / バケット1つ**
- 課金単位はバケット。複数バケット運用するなら本数分課金。
- 無料枠超過時：**ハード停止**（新規アップロード不可、既存ファイルの閲覧・DLは継続可）
- 課金後はプロバイダ上限まで容量無制限
- 上限到達前に通知（80% / 95%）
- 無料枠はアカウント単位で合算（バケット量産による回避不可）

---

## 5. セキュリティ

- E2E暗号化（ゼロ知識）
- 暗号化方式：AES-256-GCM もしくは XChaCha20-Poly1305
- 鍵導出：Argon2id（パスフレーズ → マスター鍵）
- ファイル名・パス・メタデータも暗号化
- 大ファイルはチャンク分割、各チャンクを独立に認証付き暗号化
- 鍵保管：iOS Keychain
- 回復用ニーモニックの発行＋オフライン保管を強く案内

---

## 6. 復旧保証

- **暗号化フォーマットを公開**
- アプリ非依存の復号CLIツールを別途配布
- サブスク失効・アプリ停止・運営消滅でもデータ救出可能であることを保証
- ストア説明文・LPでもこの点を前面に出す

## 12. 技術スタック・実装上の論点

### スタック
- React Native
- 暗号化：`react-native-quick-crypto` または libsodium のネイティブバインディング
- バックグラウンドアップロード：ネイティブモジュール（NSURLSession Background）または `react-native-background-upload`

### 先に潰すべき技術リスク
- JSでの暗号化はパフォーマンス上不可。ネイティブ実装が必須
- iOSのバックグラウンドアップロード制約
- 大ファイルのストリーミング暗号化（メモリに全部載せられない）

## 14. 確定事項（旧「未決事項」）

### 鍵設計
- 形式：**BIP39 12語ニーモニックのみ**
- スコープ：**アカウントに1つ**の鍵（全バケット共通）
- オンボーディング時の検証：**「書き写した」チェックボックスのみ**
- 鍵ファイルエクスポート：設定タブから後でいつでも可能（Share sheet 経由）

### 無料トライアル
- **追加トライアルは設けない**。無料枠（マネージド5GB / BYO10GB）が永続なので、これ自体が実質トライアルとして機能する。
- 課金は容量を超えそうになったタイミングで案内する（80% / 95% 通知）。

### サムネイル生成・保存
- 生成：取り込み時にネイティブ側で **長辺 512px / JPEG quality 70** に縮小（画像は `@bam.tech/react-native-image-resizer`、動画は `react-native-create-thumbnail` で 1 秒目のフレーム）。
- 保管：本体ファイルとは別鍵で AES-256-GCM 暗号化し、二段構えで保存。
  - ローカル：`CachesDirectory/thumbs/<id>`（暗号化済み、シェルからは見られない）
  - リモート：S3 上 `twins/<id>.t`（暗号化済み）
- 鍵：マスター鍵から HKDF (`info="thumb"`) で派生した別鍵を使用。本体ファイル鍵とは分離。
- 失敗時はサムネイル無しとしてフォールバック（取り込み自体は止めない）。

### BYO 接続情報の保管
- 保管先：**iOS Keychain**（`react-native-keychain` の `setGenericPassword`）。
- アクセシビリティ：`WHEN_UNLOCKED_THIS_DEVICE_ONLY`。iCloud Keychain 同期および iCloud バックアップ復元の対象外（端末から出さない）。
- スコープ：バケットごとに別 service id（`<BUCKET_SERVICE>.<bucketId>`）で保存。鍵自体（マスター鍵 / ニーモニック）とは独立に削除可能。
- 中身：`endpoint`, `region`, `bucket`, `accessKeyId`, `secretAccessKey`, ラベル等を JSON 直列化。
- 削除：アカウント全消去時、および個別バケット解除時に各 service を `resetGenericPassword` で消す。

## 付録A. 文言ガイドライン

### トーン

「やさしく日常的」。Notion / Bear のような、間口の広い、親しみのある口調。技術や仕様の正確さよりも、ユーザーが安心して使えることを優先する。

### 守るルール

- **専門用語は言い換える、または説明を添える**
  - BIP39 → 「12個の言葉」
  - ニーモニック → 「合言葉」
  - E2E暗号化 → 「鍵をかける」
  - 暗号化インデックス → 出さない
- **「です・ます」で統一**。体言止めはウェルカムなど一部のメインコピーだけに限る
- **抽象語より具体名**
  - 「ユーザー」「データ」より「あなた」「写真」「書類」「思い出」
- **命令形を避ける**
  - 「保管してください」→「しまっておいてください」
  - 「許可する」→「許可していただく」
- **防御的・弁明的な書き方を避ける**
  - 「〜にはアクセスできません」→「こちらからは見えません」
- **カッコ書きの技術注釈は外す**
  - 「検索（暗号化インデックスで実行）」→「検索」

### NGワード集

- 「ユーザー」「データ」「ファイル管理」「ヒエラルキー型」
- 「サブスク」（→「お支払い」「ご利用」）
- 「ハード停止」「上限到達」（→「いっぱい」「置けなくなる」）
- 「バックアップ」だけだと固いので、文脈で「あずかる」「しまう」も使う

## 付録B. UI文言案

### 01 / ウェルカム
> 大事なものを、安心して置ける場所。
>
> 写真も、書類も、思い出も。鍵はあなたが持っていてください。アプリがなくなっても、データは取り出せます。

### 02 / サインイン
> はじめまして
>
> Apple IDでサインインします。お支払いの管理だけに使うので、ファイルや鍵はこちらからは見えません。

チェックリスト:
- ファイルは手元で鍵をかけます
- 必要なのはApple IDだけ
- やめたいときはすぐに消せます

### 03 / バケット選択
> どこに置きますか?
>
> どちらを選んでも、安全さは変わりません。あとから変えられます。

カード:
- マネージド：すぐ使えるおまかせプラン。ストレージ料金もアプリ料金に含まれます。
- BYO：お持ちのストレージにつなぎます。ストレージ料金はご利用先にお支払いください。

### 03B / BYO接続情報
> つなぎ先を教えてください
>
> S3互換ならどこでも大丈夫です（Cloudflare R2、Wasabi、Backblazeなど）

### 04 / 回復フレーズ
> あなたの合言葉
>
> この12個の言葉が、あなたの鍵です。紙に書いて、大切な場所にしまっておいてください。なくしてしまうと、誰にも開けられなくなります。

チェックボックス:
- 書き写しました。あとからは見られないこと、わかっています。

### 05 / 写真権限
> 写真もあずかります
>
> 許可していただくと、新しい写真は鍵をかけて自動でしまっておきます。いつでもやめられます。

3項目:
- 見るだけ。書いたり消したりはしません。
- Wi-Fiのときだけ。通信量は気にしなくて大丈夫です。
- 小さな写真も鍵つきで。サッと開けるよう手元にも置いておきます。

### 06 / ツアー
> 準備できました
>
> 使うのはこの3つだけ。

タブ説明:
- フォルダ / ファイルを置く場所
- アルバム / 写真を日付で
- 設定 / 保存先と鍵のこと

### メインタブの細部

- 検索（プレースホルダ）
